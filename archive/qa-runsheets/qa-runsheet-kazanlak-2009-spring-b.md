### Source Reliability Patterns

- **O1 confirmed:** DPF scans are authoritative for unit numbers. The unit range discrepancies on Mar 25-26 appear to be extraction errors from original data entry.
- **Unit continuity:** When DPF values are used, unit ranges are continuous (each day starts where previous day ended). CSV values create overlaps.
- **Missing record:** Mar 8 data exists in DPF but was not extracted to CSV. This is a significant gap of 91 survey units.
- **Form errors:** Apr 5 DPF has Start/End Unit and Time fields swapped, but data was correctly extracted.

### Implications for Future QA

- Check for missing records by verifying unit range continuity
- DPF unit ranges should be continuous; overlapping CSV values indicate extraction errors
- Handwritten notes may have date format ambiguities (European vs US format)
